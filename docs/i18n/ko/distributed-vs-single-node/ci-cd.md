# CI/CD 및 자동 테스트

Rejourney는 GitHub Actions를 사용하여 전체 모노레포에서 코드 품질을 보장합니다. 메인 브랜치에 대한 모든 풀 요청과 푸시는 포괄적인 테스트 배터리를 트리거합니다.

## 테스트 스위트

### 1. 백엔드 API 테스트
`backend/` 디렉터리에 있는 이러한 테스트는 핵심 논리와 데이터베이스 상호 작용이 안정적인지 확인합니다.
* **린팅**: ESLint를 사용하여 코드 스타일을 적용하고 일반적인 오류를 포착합니다.
* **단위 테스트**: Vitest로 구동되며 서비스 로직, 유틸리티 기능 및 API 컨트롤러를 테스트합니다.
* **빌드 검증**: TypeScript 소스가 최종 배포판으로 올바르게 컴파일되는지 확인합니다.

### 2. React Native SDK 테스트
`packages/react-native/`에 있는 이러한 테스트는 플랫폼 간 안정성에 매우 중요합니다.
* **TypeScript 확인**: 전체 SDK에서 유형을 검증하여 잠재적인 브리지 불일치를 포착합니다.
* **린팅**: 일관된 코드 품질을 적용합니다.
* **빌드 검증**: 배포를 위해 패키지를 번들로 묶을 수 있는지 확인하기 위해 준비 스크립트를 실행합니다.

### 3. 웹 대시보드 테스트
`dashboard/web-ui/`에 위치하며 사용자 인터페이스와 SSR에 중점을 둡니다.
* **TypeScript 확인**: 경로 안전을 보장하기 위해 React Router 유형 생성을 포함합니다.
* **SSR 빌드**: 전체 Remix/React Router 애플리케이션이 서버 측 렌더링을 위해 구축될 수 있는지 확인합니다.

## 기본 통합 테스트
CI/CD의 가장 강력한 부분 중 하나는 실제 플랫폼 환경에서 SDK의 검증입니다.

### iOS 통합(macos-최신)
* **새로 설치**: CI는 처음부터 완전히 새로운 React Native 프로젝트를 만듭니다.
* **패키지 주입**: `npm pack`를 사용하여 로컬 SDK를 번들로 묶어 테스트 앱에 설치합니다.
* **CocoaPods 확인**: `pod install`를 실행하여 기본 종속성과 Podspec이 올바르게 연결되었는지 확인합니다.
* **빌드 검증**: `xcodebuild`를 실행하여 테스트 앱이 통합된 SDK로 성공적으로 컴파일되는지 확인합니다.

### Android 통합(우분투 최신)
* **새로 설치**: iOS와 유사하게 새로운 Android 기반 React Native 프로젝트가 초기화됩니다.
* **빌드 검증**: `./gradlew assembleDebug`를 실행하여 Android 네이티브 코드에 매니페스트 충돌이나 컴파일 오류가 없는지 확인합니다.

## 배포 및 게시 논리

### 자동화된 클라우드 배포(VPS)
프로덕션 환경에 대한 배포는 버전 관리를 통해 관리됩니다.
* **버전 확인**: 전용 작업은 루트 `package.json` 버전을 이전 커밋과 비교합니다.
* **조건부 트리거**: 버전이 증가한 경우에만 배포가 진행됩니다.
* **자동화된 출시**: 트리거되면 최신 K8s 매니페스트를 적용하고 모든 배포(API, 웹 및 작업자)의 롤링 다시 시작을 수행합니다.

### 자동화된 SDK 게시(NPM)
우리는 `rejourney` 패키지에 대한 원활한 게시 흐름을 유지합니다.
* **경로 감지**: `packages/react-native/` 내부의 파일이 수정되는 경우에만 트리거됩니다.
* **레지스트리 확인**: 로컬 패키지 버전을 NPM 레지스트리의 최신 버전과 비교합니다.
* **자동 게시**: 로컬 버전이 더 높으면 모든 테스트를 통과한 후 자동으로 새 버전을 NPM에 게시합니다.
