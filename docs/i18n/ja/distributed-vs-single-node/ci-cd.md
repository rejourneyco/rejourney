# CI/CD および自動テスト

Rejourney は GitHub Actions を使用して、モノリポジトリ全体のコード品質を保証します。プル リクエストとメイン ブランチへのプッシュが行われるたびに、一連の包括的なテストがトリガーされます。

## テストスイート

### 1. バックエンド API テスト
これらのテストは、`backend/` ディレクトリにあり、コア ロジックとデータベースの相互作用が安定していることを確認します。
* **糸くず**: ESLint を使用してコード スタイルを強制し、一般的なエラーをキャッチします。
* **単体テスト**: Vitest を利用し、サービス ロジック、ユーティリティ関数、および API コントローラーをテストします。
* **ビルド検証**: TypeScript ソースが最終ディストリビューションに正しくコンパイルされることを確認します。

### 2. React Native SDK テスト
これらのテストは `packages/react-native/` にあり、クロスプラットフォームの安定性にとって重要です。
* **TypeScript 小切手**: SDK 全体で型を検証し、潜在的なブリッジの不一致を検出します。
* **糸くず**: 一貫したコード品質を強制します。
* **ビルド検証**: 準備スクリプトを実行して、パッケージを配布用にバンドルできることを確認します。

### 3. Web ダッシュボードのテスト
`dashboard/web-ui/`にあり、ユーザーインターフェイスとSSRに重点を置いています。
* **TypeScript 小切手**: ルートの安全性を確保するための React Router タイプの生成が含まれています。
* **SSRビルド**: Remix/React Router アプリケーション全体がサーバー側レンダリング用に構築できることを検証します。

## ネイティブ統合テスト
CI/CD の最も堅牢な部分の 1 つは、実際のプラットフォーム環境での SDK の検証です。

### iOS 統合 (macos-最新)
* **新規インストール**: CI は、まったく新しい React Native プロジェクトを最初から作成します。
* **パッケージインジェクション**: `npm pack` を使用してローカル SDK をバンドルし、テスト アプリにインストールします。
* **CocoaPods 検証**: `pod install` を実行して、ネイティブの依存関係と podspec が正しくリンクされていることを確認します。
* **ビルド検証**: `xcodebuild` を実行して、統合された SDK でテスト アプリが正常にコンパイルされることを確認します。

### Android 統合 (ubuntu-最新)
* **新規インストール**: iOS と同様に、新しい Android ベースの React Native プロジェクトが初期化されます。
* **ビルド検証**: `./gradlew assembleDebug` を実行して、Android ネイティブ コードにマニフェストの競合やコンパイル エラーがないことを確認します。

## 導入と公開ロジック

### 自動クラウド展開 (VPS)
実稼働環境へのデプロイメントは、バージョン管理によって制限されます。
* **バージョンチェック**: 専用ジョブは、ルート `package.json` バージョンを以前のコミットと比較します。
* **条件付きトリガー**: バージョンがインクリメントされた場合にのみ、デプロイメントが続行されます。
* **自動ロールアウト**: トリガーされると、最新の K8s マニフェストが適用され、すべてのデプロイメント (API、Web、およびワーカー) のローリング再起動が実行されます。

### 自動 SDK パブリッシング (NPM)
私たちは、`rejourney` パッケージのシームレスな公開フローを維持しています。
* **パスに依存する**: `packages/react-native/` 内のファイルが変更された場合にのみトリガーされます。
* **レジストリチェック**: ローカル パッケージのバージョンを NPM レジストリの最新バージョンと比較します。
* **自動公開**: ローカル バージョンの方が高い場合、すべてのテストに合格した後、新しいバージョンが NPM に自動的に公開されます。
