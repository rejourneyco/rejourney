# CI/CD और स्वचालित परीक्षण

Rejourney संपूर्ण मोनोरेपो में कोड गुणवत्ता सुनिश्चित करने के लिए GitHub Actions का उपयोग करता है। प्रत्येक पुल अनुरोध और मुख्य शाखा को पुश करने से परीक्षणों की एक व्यापक बैटरी शुरू हो जाती है।

## टेस्ट सूट

### 1. बैकएंड API टेस्ट
`backend/` निर्देशिका में स्थित, ये परीक्षण सुनिश्चित करते हैं कि मुख्य तर्क और डेटाबेस इंटरैक्शन स्थिर हैं।
* **लिंटिंग**: कोड शैली को लागू करने और सामान्य त्रुटियों को पकड़ने के लिए ESLint का उपयोग करता है।
* **यूनिट परीक्षण**: विटेस्ट द्वारा संचालित, सेवा तर्क, उपयोगिता कार्यों और API नियंत्रकों का परीक्षण।
* **सत्यापन बनाएँ**: सुनिश्चित करता है कि TypeScript स्रोत अंतिम वितरण में सही ढंग से संकलित हो।

### 2. React Native SDK परीक्षण
`packages/react-native/` में स्थित, ये परीक्षण क्रॉस-प्लेटफ़ॉर्म स्थिरता के लिए महत्वपूर्ण हैं।
* **TypeScript जांचें**: संभावित ब्रिज बेमेल को पकड़ते हुए, संपूर्ण SDK में प्रकारों को मान्य करता है।
* **लिंटिंग**: सुसंगत कोड गुणवत्ता लागू करता है।
* **सत्यापन बनाएँ**: वितरण के लिए पैकेज को बंडल किया जा सकता है यह सुनिश्चित करने के लिए तैयार स्क्रिप्ट चलाता है।

### 3. वेब डैशबोर्ड टेस्ट
उपयोगकर्ता इंटरफ़ेस और SSR पर ध्यान केंद्रित करते हुए, `dashboard/web-ui/` में स्थित है।
* **TypeScript जांचें**: मार्ग सुरक्षा सुनिश्चित करने के लिए रिएक्ट राउटर प्रकार की पीढ़ी शामिल है।
* **एसएसआर बिल्ड**: सत्यापित करता है कि संपूर्ण रीमिक्स/रिएक्ट राउटर एप्लिकेशन सर्वर-साइड रेंडरिंग के लिए बनाया जा सकता है।

## मूल एकीकरण परीक्षण
हमारे CI/CD के सबसे मजबूत हिस्सों में से एक वास्तविक प्लेटफ़ॉर्म वातावरण पर SDK का सत्यापन है।

### iOS एकीकरण (मैकओएस-नवीनतम)
* **ताज़ा इंस्टाल**: CI बिल्कुल नया React Native प्रोजेक्ट बनाता है।
* **पैकेज इंजेक्शन**: यह `npm pack` का उपयोग करके स्थानीय SDK को बंडल करता है और इसे परीक्षण ऐप में इंस्टॉल करता है।
* **CocoaPods सत्यापन**: मूल निर्भरता और पॉडस्पेक्स सही ढंग से जुड़े हुए हैं यह सुनिश्चित करने के लिए `pod install` चलाता है।
* **सत्यापन बनाएँ**: यह सुनिश्चित करने के लिए `xcodebuild` निष्पादित करता है कि परीक्षण ऐप SDK एकीकृत के साथ सफलतापूर्वक संकलित हो।

### Android एकीकरण (उबंटू-नवीनतम)
* **ताज़ा इंस्टाल**: iOS के समान, एक नया Android-आधारित React Native प्रोजेक्ट आरंभ किया गया है।
* **सत्यापन बनाएँ**: यह सुनिश्चित करने के लिए `./gradlew assembleDebug` चलाता है कि Android मूल कोड में कोई स्पष्ट विरोध या संकलन त्रुटियां नहीं हैं।

## परिनियोजन एवं प्रकाशन तर्क

### स्वचालित क्लाउड परिनियोजन (VPS)
हमारे उत्पादन परिवेश में परिनियोजन संस्करणीकरण द्वारा निर्धारित होता है।
* **संस्करण की जाँच करें**: एक समर्पित कार्य रूट `package.json` संस्करण की तुलना पिछले कमिट से करता है।
* **सशर्त ट्रिगर**: परिनियोजन तभी आगे बढ़ता है जब संस्करण बढ़ाया गया हो।
* **स्वचालित रोलआउट**: यदि ट्रिगर किया जाता है, तो यह नवीनतम K8s मैनिफ़ेस्ट लागू करता है और सभी परिनियोजन (एपीआई, वेब और वर्कर्स) का रोलिंग पुनरारंभ करता है।

### स्वचालित SDK प्रकाशन (एनपीएम)
हम `rejourney` पैकेज के लिए एक निर्बाध प्रकाशन प्रवाह बनाए रखते हैं।
* **पथ संवेदनशील**: केवल तभी ट्रिगर होता है जब `packages/react-native/` के अंदर फ़ाइलें संशोधित की जाती हैं।
* **रजिस्ट्री जांच**: NPM रजिस्ट्री पर नवीनतम संस्करण के विरुद्ध स्थानीय पैकेज संस्करण की तुलना करता है।
* **ऑटो प्रकाशित**: यदि स्थानीय संस्करण उच्चतर है, तो यह सभी परीक्षण पास होने के बाद स्वचालित रूप से NPM पर नया संस्करण प्रकाशित करता है।
