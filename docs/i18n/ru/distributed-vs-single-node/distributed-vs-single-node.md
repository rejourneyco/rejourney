# Распределенное и одноузловое облако

Rejourney поддерживает две официальные формы самостоятельного развертывания:

- **Одноузловой Docker Compose** для одного сервера или VPS
- **Распределенный K3s** для производственных кластеров и горизонтального масштабирования

Оба теперь используют одну и ту же базовую модель бэкэнда:

- конечные точки хранилища поддерживаются базой данных
- входящие загрузки проходят через серверную ретрансляцию загрузки
- рабочие обрабатывают проверенные артефакты
- видимость повтора зависит от артефактов

---

## Сравнение функций

| Особенность | Распределенное облако | Одноузловое облако |
|---------|--------------------|-------------------|
| Платформа | K3s | Docker Compose |
| Масштаб | Многоузловой | Одноузловой |
| Общественные входы | Traefik вход | Traefik контейнер |
| Путь загрузки | API + служба загрузки и загрузки | API + служба загрузки и загрузки |
| Хранилище источника истины | `storage_endpoints` стол | `storage_endpoints` стол |
| Хранилище объектов по умолчанию | Внешний S3 | Встроенный MinIO |
| Внешняя поддержка S3 | Да | Да |
| Секретное шифрование | `STORAGE_ENCRYPTION_KEY` | `STORAGE_ENCRYPTION_KEY` |
| Порядок обновления | k8s развертывание + задания | `deploy.sh update` |

---

## Модель общего хранилища

В обеих моделях развертывания конфигурация хранилища времени выполнения берется из Postgres, а не из резервной среды env.

Это означает:

- конечная точка хранилища активных объектов хранится в `storage_endpoints`
- секретные ключи доступа зашифрованы в `key_ref`
- среда выполнения считывает строку базы данных
- Скрипты начальной загрузки/установки отвечают за синхронизацию ввода `.env` со строкой базы данных.

Это делает самостоятельную версию Docker гораздо ближе к рабочей версии и local-k8s, чем к старой резервной модели.

---

## Когда следует выбирать одноузловой Docker Compose

Выбирайте Docker Compose, когда:

- вы выполняете развертывание на одном хосте VPS или на «голом железе»
- вам нужен самый быстрый путь установки
- вам нужен встроенный MinIO по умолчанию
- вам не требуется многоузловое масштабирование или операции Kubernetes.

Официальные точки входа:

- `docker-compose.selfhosted.yml`
- `scripts/selfhosted/deploy.sh`
- `docs/selfhosted/README.md`

---

## Когда следует выбирать распределенную версию K3s

Выбирайте K3s, когда:

- вам нужно несколько узлов
- вам нужны Kubernetes-родные операции и секретная обработка
- вы хотите независимо масштабировать API, загружать и рабочие сервисы
- вам нужны последовательные развертывания и более сильная инфраструктурная изоляция

Путь K3s находится под `k8s/` и `scripts/k8s/`.

---

## Операционная разница

Основное отличие больше не в модели данных. Это рабочая форма:

- Compose: одна машина, одна сеть Docker, один сценарий оператора.
- K3s: несколько модулей, пространства имен, вход в кластер, задания и секреты Kubernetes.

---

## Практическое руководство

Начните с одного узла Compose, если вы хотите быстро разместить его самостоятельно.

Перейдите на K3s, когда вам нужно:

- большая пропускная способность
- развертывание скользящего кластера
- горизонтальное масштабирование
- более устойчивое разделение инфраструктуры
