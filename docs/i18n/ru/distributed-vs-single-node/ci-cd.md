# CI/CD и автоматическое тестирование

Rejourney использует GitHub Actions для обеспечения качества кода во всем монорепозитории. Каждый запрос на включение и отправку в основную ветку запускает комплексную батарею тестов.

## Тестовые наборы

### 1. Бэкэнд-тесты API
Эти тесты, расположенные в каталоге `backend/`, обеспечивают стабильность базовой логики и взаимодействия с базой данных.
* **Линтинг**: использует ESLint для обеспечения соблюдения стиля кода и выявления распространенных ошибок.
* **Модульные тесты**: на базе Vitest, тестирование сервисной логики, служебных функций и контроллеров API.
* **Проверка сборки**: обеспечивает правильную компиляцию исходного кода TypeScript в окончательный дистрибутив.

### 2. React Native SDK Тесты
Эти тесты, расположенные в `packages/react-native/`, имеют решающее значение для межплатформенной стабильности.
* **TypeScript Проверить**: проверяет типы по всему SDK, выявляя потенциальные несоответствия моста.
* **Линтинг**: обеспечивает стабильное качество кода.
* **Проверка сборки**: запускает сценарий подготовки, чтобы убедиться, что пакет можно собрать для распространения.

### 3. Тесты веб-панели
Расположен в `dashboard/web-ui/` и ориентирован на пользовательский интерфейс и SSR.
* **TypeScript Проверить**: включает генерацию типа маршрутизатора React для обеспечения безопасности маршрута.
* **Строительство ССР**: проверяет, может ли все приложение Remix/React Router быть создано для рендеринга на стороне сервера.

## Нативное интеграционное тестирование
Одной из наиболее надежных частей нашего CI/CD является проверка SDK на реальных платформах.

### iOS Интеграция (последняя версия MacOS)
* **Свежая установка**: CI создает совершенно новый проект React Native с нуля.
* **Инъекция пакета**: он объединяет локальный SDK с помощью `npm pack` и устанавливает его в тестовое приложение.
* **CocoaPods Проверка**: запускает `pod install`, чтобы убедиться в правильности связывания собственных зависимостей и подспецификаций.
* **Проверка сборки**: выполняет `xcodebuild`, чтобы гарантировать успешную компиляцию тестового приложения с интегрированным SDK.

### Android Интеграция (последняя версия Ubuntu)
* **Свежая установка**: аналогично iOS, инициализируется новый проект React Native на основе Android.
* **Проверка сборки**: запускает `./gradlew assembleDebug`, чтобы убедиться в отсутствии конфликтов манифеста или ошибок компиляции в собственном коде Android.

## Логика развертывания и публикации

### Автоматическое развертывание в облаке (VPS)
Развертывание в нашей производственной среде ограничивается контролем версий.
* **Проверка версии**: специальное задание сравнивает корневую версию `package.json` с предыдущей фиксацией.
* **Условный триггер**: развертывание продолжается только в том случае, если версия была увеличена.
* **Автоматизированное внедрение**: при срабатывании применяются последние манифесты K8s и выполняется последовательный перезапуск всех развертываний (API, Интернет и рабочие процессы).

### Автоматизированная публикация SDK (NPM)
Мы поддерживаем бесперебойный процесс публикации пакета `rejourney`.
* **Чувствителен к пути**: срабатывает только при изменении файлов внутри `packages/react-native/`.
* **Проверка реестра**: сравнивает версию локального пакета с последней версией в реестре NPM.
* **Автоматическая публикация**: если локальная версия выше, она автоматически публикует новую версию в NPM после прохождения всех тестов.
