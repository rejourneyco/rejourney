# CI/CD والاختبار الآلي

يستخدم Rejourney GitHub Actions لضمان جودة الكود عبر monorepo بأكمله. يؤدي كل طلب سحب ودفع إلى الفرع الرئيسي إلى إطلاق مجموعة شاملة من الاختبارات.

## أجنحة الاختبار

### 1. اختبارات الواجهة الخلفية API
تقع هذه الاختبارات في دليل `backend/`، وتضمن استقرار المنطق الأساسي وتفاعلات قاعدة البيانات.
* **فحص**: يستخدم ESLint لفرض نمط التعليمات البرمجية والتعرف على الأخطاء الشائعة.
* **اختبارات الوحدة**: مدعوم من Vitest واختبار منطق الخدمة ووظائف الأداة المساعدة ووحدات التحكم API.
* **التحقق من البناء**: يضمن تجميع مصدر TypeScript بشكل صحيح في التوزيع النهائي.

### 2. اختبارات React Native SDK
تقع هذه الاختبارات في `packages/react-native/`، وهي ضرورية لتحقيق الاستقرار عبر الأنظمة الأساسية.
* **فحص TypeScript**: التحقق من صحة الأنواع عبر SDK بالكامل، مع اكتشاف عدم تطابق الجسر المحتمل.
* **فحص**: يفرض جودة رمز متسقة.
* **التحقق من البناء**: يقوم بتشغيل البرنامج النصي للتحضير لضمان إمكانية تجميع الحزمة للتوزيع.

### 3. اختبارات لوحة تحكم الويب
يقع في `dashboard/web-ui/`، مع التركيز على واجهة المستخدم وSSR.
* **فحص TypeScript**: يتضمن إنشاء نوع React Router لضمان سلامة المسار.
* **بناء SSR**: التحقق من إمكانية إنشاء تطبيق Remix/React Router بالكامل للعرض من جانب الخادم.

## اختبار التكامل الأصلي
أحد أقوى الأجزاء في CI/CD هو التحقق من صحة SDK في بيئات النظام الأساسي الحقيقية.

### تكامل iOS (نظام MacOS-الأحدث)
* **تثبيت جديد**: يقوم CI بإنشاء مشروع React Native جديد تمامًا من الصفر.
* **حقن الحزمة**: يقوم بتجميع SDK المحلي باستخدام `npm pack` وتثبيته في تطبيق الاختبار.
* **التحقق من CocoaPods**: تشغيل `pod install` لضمان ربط التبعيات الأصلية ومواصفات podspec بشكل صحيح.
* **التحقق من البناء**: يتم تنفيذ `xcodebuild` لضمان تجميع تطبيق الاختبار بنجاح مع دمج SDK.

### تكامل Android (الأحدث لنظام أوبونتو)
* **تثبيت جديد**: على غرار iOS، تتم تهيئة مشروع React Native الجديد المستند إلى React Native.
* **التحقق من البناء**: يقوم بتشغيل `./gradlew assembleDebug` للتأكد من عدم وجود تعارضات واضحة أو أخطاء في الترجمة في الكود الأصلي Android.

## منطق النشر والنشر

### النشر السحابي الآلي (VPS)
يتم النشر في بيئة الإنتاج الخاصة بنا عن طريق الإصدار.
* **التحقق من الإصدار**: مهمة مخصصة تقارن إصدار `package.json` الجذري مع الالتزام السابق.
* **الزناد الشرطي**: يستمر النشر فقط في حالة زيادة الإصدار.
* **الطرح الآلي**: في حالة تشغيله، فإنه يطبق أحدث بيانات K8s وينفذ إعادة تشغيل متجددة لجميع عمليات النشر (واجهة برمجة التطبيقات والويب والعاملين).

### النشر الآلي SDK (NPM)
نحن نحافظ على تدفق النشر السلس لحزمة `rejourney`.
* **حساس للمسار**: يتم تشغيله فقط عندما يتم تعديل الملفات الموجودة داخل `packages/react-native/`.
* **فحص التسجيل**: يقارن إصدار الحزمة المحلية مع الإصدار الأحدث الموجود في سجل NPM.
* **النشر التلقائي**: إذا كان الإصدار المحلي أعلى، فإنه ينشر الإصدار الجديد تلقائيًا إلى NPM بعد اجتياز جميع الاختبارات.
