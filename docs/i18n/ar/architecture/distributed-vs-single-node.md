# الموزعة مقابل السحابة ذات العقدة الواحدة

يدعم Rejourney شكلين رسميين للنشر ذاتي الاستضافة:

- **عقدة واحدة Docker Compose** لخادم واحد أو VPS
- **توزيع K3s** لمجموعات الإنتاج والقياس الأفقي

كلاهما يستخدم الآن نفس نموذج الواجهة الخلفية الأساسية:

- نقاط نهاية التخزين مدعومة بقاعدة البيانات
- يتم استيعاب التحميلات من خلال ترحيل التحميل المملوك للواجهة الخلفية
- يقوم العمال بمعالجة القطع الأثرية التي تم التحقق منها
- رؤية إعادة التشغيل تعتمد على القطع الأثرية

---

## مقارنة الميزات

| ميزة | السحابة الموزعة | سحابة أحادية العقدة |
|---------|--------------------|-------------------|
| منصة | K3s | Docker Compose |
| مقياس | متعدد العقدة | عقدة واحدة |
| المداخل العامة | دخول Traefik | حاوية Traefik |
| مسار التحميل | API + خدمة الاستيعاب والتحميل | API + خدمة الاستيعاب والتحميل |
| مصدر تخزين الحقيقة | جدول `storage_endpoints` | جدول `storage_endpoints` |
| تخزين الكائنات الافتراضية | خارجي S3 | المدمج في MinIO |
| دعم خارجي S3 | نعم | نعم |
| التشفير السري | `STORAGE_ENCRYPTION_KEY` | `STORAGE_ENCRYPTION_KEY` |
| تحديث التدفق | نشر k8s + الوظائف | `deploy.sh update` |

## نموذج التخزين المشترك

في كلا نموذجي النشر، يأتي تكوين تخزين وقت التشغيل من Postgres، وليس من خيار env الاحتياطي.

وهذا يعني:

- يتم تخزين نقطة نهاية تخزين الكائنات النشطة في `storage_endpoints`
- يتم تشفير مفاتيح الوصول السرية إلى `key_ref`
- يقرأ وقت التشغيل صف قاعدة البيانات
- تكون البرامج النصية للتمهيد/التثبيت مسؤولة عن مزامنة إدخال `.env` في صف قاعدة البيانات

وهذا يجعل Docker المستضاف ذاتيًا أقرب بكثير إلى المنتج وlocal-k8s من النموذج الاحتياطي القديم.

## متى تختار العقدة المفردة Docker Compose

اختر Docker Compose عندما:

- أنت تقوم بالنشر إلى VPS أو مضيف معدني
- تريد أسرع مسار التثبيت
- تريد MinIO مدمجًا بشكل افتراضي
- لا تحتاج إلى تحجيم متعدد العقد أو عمليات Kubernetes

نقاط الدخول الرسمية:

- `docker-compose.selfhosted.yml`
- `scripts/selfhosted/deploy.sh`
- `docs/selfhosted/README.md`

---

## متى يتم اختيار K3s الموزعة

اختر K3s عندما:

- تحتاج إلى عقد متعددة
- تريد عمليات Kubernetes الأصلية والتعامل السري
- تريد توسيع نطاق خدمات API والتحميل وخدمات العاملين بشكل مستقل
- تريد عمليات نشر متجددة وعزلًا أقوى للأشعة تحت الحمراء

يقع مسار K3s ضمن `k8s/` و`scripts/k8s/`.

---

## الفرق التشغيلي

الفرق الرئيسي لم يعد نموذج البيانات بعد الآن. وهو الشكل التشغيلي:

- Compose: جهاز واحد، شبكة Docker واحدة، برنامج نصي لمشغل واحد
- K3s: وحدات متعددة ومساحات الأسماء ودخول المجموعة ووظائف وأسرار Kubernetes

---

## إرشادات عملية

ابدأ بعقدة واحدة Compose إذا كنت تريد الاستضافة الذاتية بسرعة.

انتقل إلى K3s عندما تحتاج إلى:

- المزيد من الإنتاجية
- نشر الكتلة المتداول
- التحجيم الأفقي
- فصل البنية التحتية بشكل أكثر مرونة
